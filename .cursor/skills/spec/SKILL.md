# Skill: Spec Decision → Implementation Plan

## 目的

- 仕様・設計の論点を分解し、選択肢を比較して推奨案と実装方針を提示する
- **判断はユーザーが行う**。AI は判断材料を揃える
- 共通概念は [../_shared/analysis-concepts.md](../_shared/analysis-concepts.md) を参照
- Clear Bid 仕様は `docs/clear-bid-spec-v1.0.pdf` / `clear-bid-domain.mdc` を優先

## 入力情報

- [テーマ] / [現行] / [問題]
- [制約]
- [関連画面]
- SEARCH_SCOPE

## 手順

### Step 1: 現状把握
1. 近い既存機能を Read し、現行の構造・パターンを把握する
2. shared 型・API・画面のデータフローを追跡する

### Step 2: 論点分解と比較
3. 論点を最大7つに分解する
4. 各論点に A/B 案とメリデメ
5. 推奨案と理由3つ（既存整合 / 性能 / 運用）

### Step 3: 実装計画
6. 候補ファイル / DB影響 / 共通再利用
7. 受入条件を Given/When/Then
8. ユーザーが決めるべきことを1行質問にする
9. 影響範囲とテスト計画

## 注意

- 確信度80%未満は断定しない
- Access / 本番 D1 / 外部 AI 本接続は後回し領域。勝手に実装計画の前提にしない（必要なら別枠で提案）

## 出力フォーマット

1. 論点（最大7）
2. 選択肢A/B
3. 推奨案（理由3つ）
4. 実装方針
5. 受入条件（Given/When/Then）
6. 決めるべき1行質問
7. 影響範囲/テスト計画
8. 自己検証
