# Skill: Incident Investigation → Minimal Fix

バグ修正との違い: **ログ・スタックトレースが起点**で、再現手順が不明なことが多い。
共通概念は [../_shared/analysis-concepts.md](../_shared/analysis-concepts.md) を参照。

## 入力情報

- [タイトル] / [現象] / [期待] / [再現条件]
- [ログ]
- [補足]（直近変更 / PR）
- SEARCH_SCOPE

## 手順

1. ログから発生箇所を特定
2. 直近変更との相関を確認
3. 原因候補を P1〜P3 に絞る
4. P1 確定の最小追加確認を提示
5. 確信度80%以上なら最小修正と再発防止を提案
6. 地雷チェック

## 出力フォーマット

1. 関連ファイル
2. 原因候補（P1〜P3）
3. 追加確認
4. 検証手順
5. 修正方針
6. パッチ案（80%以上のみ）
7. テスト
8. 影響範囲
9. 自己検証
10. 地雷チェック
